OK, now really reviewing this.

1. I am not too happy with the unstructured `src/doc/en/reference/misc/index.rst`, for example, why aren't `cython`, `cython_c` and `inline_fortran` together, what is `sage/ext/memory` doing there...? Perhaps you're simply trying to lump too many things together.

5. I would like to see `sage/misc/typecheck` and `sage/structure/element_verify` removed (unused modules)

7. I think we can remove the `WRONG (for now) docs` which have been wrong for over 6 years...

8. Many new `INPUT` blocks are badly formatted, the correct formatting is

```
INPUT:

- ``metadata`` -- The metadata_by_opname from a wrapper module
```

9. I think this comment can be removed:

```
TODO:

This stuff probably belongs somewhere like sage/misc/misc_c.pyx.
The only reason it's in its own file here is because of some circular cimport
problems (can't cimport Integer to sage/misc/misc_c.pyx). For further
discussion see
http://codespeak.net/pipermail/cython-dev/2008-February/000057.html
```

10. Don't touch modules removed in #18180.

21. `"Nested lists"` -> `"Flatten nested lists"`
