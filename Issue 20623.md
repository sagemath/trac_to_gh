# Issue 20623: sage_getargspec on Cython class instances prefers __init__ over __call__

Issue created by migration from Trac.

Original creator: jdemeyer

Original creation time: 2016-06-21 08:35:30

CC:  simonking nthiery

The following Cython code


```
cimport cython
`@`cython.embedsignature(True)
cdef class MyClass(object):
    def __init__(self, x):
        pass
        
    def __call__(self, y):
        pass
        
from sage.misc.sageinspect import sage_getargspec
print(sage_getargspec(MyClass("hello")))
```


gives the argspec of `__init__` instead of `__call__`:


```
ArgSpec(args=['x'], varargs=None, keywords=None, defaults=None)
```
