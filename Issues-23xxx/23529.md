<a id='comment:4'></a>Just to start.  An example from the code that I am now looking at:

```
def _richcmp__(self, other, op):
    """
    Compare this ideal with the other ideal with respect to ``op``.
    ...
    """
    if self.ring() != other.ring():
        raise ValueError("rings must be the same")

    return richcmp(self.module(), other.module(), op)  
```

Feel free to suggest a better idiom.

<a id='comment:7'></a>OK, so let me comment. It is important to know that `_richcmp_` guarantees (through the coercion model) that both arguments have the same parent. For ideals, I would guess that this implies that the rings are the same. In that case, the check

```
if self.ring() != other.ring():
    raise ValueError("rings must be the same")
```
is redundant.
