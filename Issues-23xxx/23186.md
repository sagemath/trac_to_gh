<a id='comment:5'></a>Because to do something at the category level, you should be doing something for all objects in that category, not for one specific object (in a specific implementation). Now, if you wanted to have something in `Rings` checking if a morphism to `ZZ` was surjective returning `True`, then that would be good because it is true for all rings (of characteristic 0). Although `map` has a concrete implementation of `is_surjective`, which is a technical detail that would need to be worked around (which that method probably should be lifted to the category of set morphisms).

The fix LGTM.
