<a id='comment:1'></a>The attached patch fixes the issue. The problem was that we were copying the files at one point in the code, and then running Cython later. Of course, if Cython failed, the code was still copied, which meant it would pass a timestamp comparison on later builds. The patch fixes this by only copying once the file is successfully built. 

In addition, it slightly expands the capabilities of William's parallel build code: now, in addition to accepting strings, it accepts pairs of the form `[f, v]`, and calls `f(v)`. The previous code did this with `f` always being `run_command`.
