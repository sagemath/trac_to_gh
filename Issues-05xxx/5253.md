<a id='comment:4'></a>FYI, the special casing is because there is no good way right now to represent a function from R<sup>n</sup> to R<sup>m</sup>.  I think people would naturally use either a list, tuple, or vector of symbolic expressions.  The Matrix special case is so that you can do nested jacobians to compute the hessian matrix, like what is illustrated in the doctests.
