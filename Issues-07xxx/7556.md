This is inconsistent:

```
sage: (9/2).round()
4
sage: RDF('4.5').round()
5
sage: import __builtin__
sage: __builtin__.round(float('4.5'))
5.0
sage: RR('4.5').round()
5
```

It's also inconsistent for negatives.  The simple fix is to make the default rounding direction 'away' in rational.pyx.
