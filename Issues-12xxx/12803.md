This is a bug in Singular 3.1.7 (fixed in 4.0.1, see ticket #17254):

```
R.<xe,xv> = ZZ[]

J = Ideal(4*xv^3+3*xv^2, 3*xe*xv^2+xe-2*xv);

I  = Ideal(-3, -3*xv-1, -3);
I2 = Ideal(-3, -3*xv-1);
assert( I  == I2 )

Q1 = J.quotient(I) # wrong
Q2 = J.quotient(I2)

assert( Q1 == Q2 ) # fails
```
