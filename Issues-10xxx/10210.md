Might be true. Nevertheless, then even generating a *single* image file for every used math character (or short math-typeset identifier) takes a big amount of time, and btw. looks pretty ugly for e.g. *p-adic* in the HTML documentation. The ratio build:doc:ptestlong is about 3:2:3 on my newer machines; on older machines ptestlong takes longer.)
