As reported by ncalexan on IRC:

```
sage: bool(x == x)
True
sage: bool(x != x)
True
sage: bool(x > x)
True
```

This appears to be caused by `SymbolicEquation.__nonzero__()` assuming in various places that the operator of the equation is ==.
