So it is something with how comparisons are done in the ring extensions. The only thing I can really guess for why removing `Rational._cmp_` breaks things is because that is being called because of the `RingExtensionElement.__getattr__` that pulls methods from the `_backend`, which then gets `Rational._cmp_` that overrides the `RingExtensionElement._richcmp_` call.
