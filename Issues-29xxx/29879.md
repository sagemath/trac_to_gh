This ticket aims to improve the interface connecting the Kenzo Program (https://www-fourier.ujf-grenoble.fr/~sergerar/Kenzo/) and [SageMath](../wiki/SageMath) (see ticket #27880 for an updated version of the interface).

In order to use Kenzo functionalities directly as methods of Sage objects, the module `Simplicial sets` can be modified to, once a simplicial set in Sage is constructed, create its analogous Kenzo simplicial set (whenever it is possible).

This belongs to the GSoC 2020 [Project](https://summerofcode.withgoogle.com/projects/#4580455350796288) by Julián Cuevas-Rozo, mentored by Miguel Marco.

<div id="comment:16" align="right">comment:16</div>

Until now, we have added the **`_kenzo_repr`** attribute to the different classes concerning simplicial sets and some related constructors. This attribute allows the automatic construction of the equivalent `KenzoObject` to a simplicial set, once it is created in [SageMath](../wiki/SageMath).

In addition, we have created classes for abstract simplexes and combinations (Kenzo objects) in order to improve the communication between the Kenzo system and [SageMath](../wiki/SageMath).

Our next step is adding the **`_kenzo_repr`** attribute for simplicial set morphisms, which makes it possible to construct pushouts and therefore other constructors of simplicial sets.
