<a id='comment:7'></a>In an ideal world we shouldn't special case debug builds. Its just that some commandline tests call third-party code that end up coughing up the default-ignored warnings. Really e.g. IPython should be fixed to not have warnings, but until that is the case we need a way to ignore them. So the `pydebug_ignore_warnings` argument intentionally looks ugly since we shouldn't use it.
