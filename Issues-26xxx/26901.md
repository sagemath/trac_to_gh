I suggest that you base #26001 on #22626 (and #26856), or at least be prepared that they might make you do extra work if you stick with (lib)GAP 4.8.6 for the time being. In particular there are few (low-level) API changes for libGAP in Sage done in #22626.

As well, perhaps more importantly, there is sometimes extra work to be done in porting to libGAP just because some parts of Sage are still pexpect-GAP based, in particular PermutationGroup (see e.g. #26889) for such an example.

We should plan to take care of this soon, perhaps even before #26001 (but not before  #22626 and #26856).
