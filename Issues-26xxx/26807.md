<a id='comment:5'></a>It's better because it allows Python subclasses to override `_mul_` and have it picked up by the coercion system.  Moreover, if you use `cdef` in a superclass but override it with `cpdef` (and possibly also `def`), there was a nasty bug in Cython that caused segfaults because Cython tried to call a non-existent function.  That's the source of the warning that Jeroen referred to.

I haven't looked at `MonoidElement`, but I would guess that the `cdef` `_mul_` there should be changed to `cpdef`.
