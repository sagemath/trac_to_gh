> Replying to [embray](#comment%3A5):
> > So it seems that maybe there are some header files missing from the distribution that are necessary in order to be able to compile cython code.

> 
> So how do you compile Python in the first place without `<crypt.h>`?

There are separate build and runtime environments for the Windows binary.  Not all the build dependencies are needed at runtime, so what ends up in the installer is a smaller runtime environment.  Granted, it's not *that* much smaller, since there is functionality in Sage that requires being able to compile things, and it seems I somehow dropped a couple devel packages from the runtime environment. 

I never had them listed explicitly as runtime dependencies, so it's possible they used to be listed as dependencies of some other package, but that dependency chain was broken.
