<a id='comment:3'></a>Positive review on the two patches for the Debian dist directory, but I will not apply ntl-forget-version.patch since that results in libntl.so and libntl-5.4.2.so being identical copies. There is a special option for cp on Linux [-d] that preserves the link, but this might break on Cygwin for example. Since updating NTL is rare we can certainly deal with changing the so name on occasion.

Cheers,

Michael
