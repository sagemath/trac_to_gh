```
sage: [prime_pi(n) for n in [500508..500510]]
[41580, 41581, 41581]
```

The problem lies with pari/gp:

```
sage: %gp

  --> Switching to GP/PARI interpreter <-- 

''
gp: for(n=500508, 500510, print(primepi(n)))

41580
45056
  *** primepi: not enough precomputed primes, need primelimit ~ 500510.
```
