# Issue 34043: RealSet: deprecate using a, b to construct the open interval (a, b) in union and intersection methods

Issue created by migration from https://trac.sagemath.org/ticket/34280

Original creator: yzh

Original creation time: 2022-08-04 22:10:06

CC:  mkoeppe

Before:

```
sage: s1 = RealSet(0,2)
sage: s1.union(1,3)
(0, 3)
```

After, `s1.union(1,3)` is no longer allowed. 
Must be written as
`sage: s1.union((1,3))` or `s1.union(RealSet(1,3))` or `s1.union(RealSet.open(1,3))`, etc.
